Creating an Efficient API Endpoint

The goal of this article is to showcase the ability to create and manage APIs effectively, and then explore how to protect and publish the API endpoint for secure and efficient usage.

As mentioned, not implementing any API protection measures and allowing direct access to the API endpoint can have negative impacts on the website's traffic, user experience, SEO ranking, and data security. Therefore, it is crucial to implement certain API security protection measures to ensure data security and maintain a good website user experience.

Implementing API Protection Measures (API Provider ‘s side)

Once the API is set up, there are several crucial protection measures that need to be put in place to ensure the security and reliability of the API:

  1. Rate Limiting: The API implements rate limiting, which restricts the maximum number of requests per minute for each IP address. This helps prevent API abuse or DDoS attacks.
  2. IP Whitelist: The API allows the definition of an IP whitelist, where specific IP addresses are exempt from the rate limiting restrictions. This is a great way to provide unlimited access to certain clients or services.
  3. CORS Restrictions: The API utilizes the CORSMiddleware to limit the allowed origins. This helps prevent unauthorized cross-origin requests.
  4. Expired Request Cleanup:  The API has a background task that periodically cleans up expired request records (e.g., 1 minute old). This ensures that the rate limiting system remains up-to-date and accurate.
  5. API Key Authentication: The API should require an API key when inserting or viewing the data, ensuring access is only granted to authorized users.
  6. Environment Variables: Sensitive variables, such as API keys and database credentials, should be stored in a .env file. This is a common practice to protect sensitive information and prevent them from being exposed in the codebase.

Integrating with a Frontend Platform (User client’s side)

To provide a user-friendly interface for viewing the API data in a table format, we’ll be using a WordPress (WP) platform as the frontend. However, we won’t be directly fetching the API endpoint from the WP platform. Instead, we’ll be using a middleware component, the WP REST API, to handle the interaction between the frontend and the backend API. This part is linked to the development of plugin, as it involves creating the Search & Filter webpage by plugin functions. 

The process would be as follows:

  1. The frontend collects the query keyword from the user input.
  2. The frontend triggers an AJAX function that sends the request to the WP REST API component.
  3. The WP REST API component then redirects the request to the initial API endpoint.
  4. The frontend receives the API response and updates the HTML elements accordingly.
  5. The process includes sorting and offset functionality.

Advantages of Using a Middleware Component

  1. Hiding the API Endpoint: By using the WP REST API as a middleware, you can hide the original API endpoint and keep the codebase secure.
  2. Request Authentication: The WP REST API component will handle the request authentication, ensuring that only authorized users can access the API.
  3. Security Features: The WP REST API component provides additional security features, such as nonce generation (to prevent CSRF), referral source checking, API key validation, and URL type conversion (to prevent SQL injection).

These security measures ensure that all audience members access the API through the normal process of visiting the WP page, providing a secure and SEO-friendly experience.
