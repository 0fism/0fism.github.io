Side-Project Intro

In today’s globalized world, having a multilingual website is crucial for reaching a wider audience. We will explore the process of creating a multilingual website using the Django framework. Django provides powerful i18n (internationalization) features that allow us to easily handle translations. Moreover, we will discuss deploying the website on an AWS instance and optimizing its SEO using technical best practices.

Tech Stack

The following full-stack technologies and essential tools were manipulated:

  1. AWS Server (EC2): The web app was deployed on an EC2, with the firewall and inbound rules configured to ensure secure access.

  1. Linux CLI: The Linux command line interface was utilized for server management and dependencies.
  2. LiteSpeed Environment: A LiteSpeed web server environment was set up.
  3. Domain DNS Configuration: The domain DNS settings were configured to point to the EC2 IP. (Not using Cloudflare)
  4. Deployment Automation: The deployment process was automated using Git commands and a .sh file, ensuring a reproducible environment setup with the same configuration and requirements.txt file.
  5. Django: The Django web framework was the backbone of this project.
  6. ORM Database Design: The database models were designed using Object-Relational Mapping (ORM) capabilities with SQLite, fulfilling the project’s requirement.
  7. Security: HTTPS, Form validation, CSRF, and SQL injection protection were implemented.

Technical Implementation

For the frontend, a reference template was used instead of starting from scratch, so without a Figma prototype. The focus of this project was on showcasing the backend abilities and functionality. Some of the key frontend and CMS features include:

  1. Language Toggle: A client-side solution for switching the language code was implemented, ensuring SEO-friendly URLs and dynamic rendering without the need for SSR.
  2. Multilingual Website: Django’s internationalization (i18n) features, along with the Rosetta library, were leveraged.
  3. Dynamic Menu and Banner: The website’s menu, banner, and company information can be managed and controlled through the CMS.
  4. Product CRUD Module: The project includes functionality for Creating, Reading, Updating, and Deleting product data.
  5. Contact Form Integration: The website features a POST contact form that stores the submitted data into the CMS.
  6. Social Media Integration: A Facebook Fan Page was embedded, and WhatsApp functionality was added to the website.
  7. Translatable Content: The majority of the website’s text content can be managed and translated through the CMS.
  8. Multi-Image Upload: The CMS supports the upload of multiple images, providing flexibility for content management.
  9. Rich Text Editor Integration: A rich text editor was integrated to allow for styled and formatted paragraphs in the CMS and display them directly in the Frontend.
  10. Mobile Responsiveness: The frontend template was fine-tuned to ensure a seamless mobile experience.

SEO Optimization

To optimize our website for search engines, we implement several technical SEO techniques. In the models.py file, we create meta titles and descriptions for each page to improve search engine visibility. Additionally, we leverage Django’s ORM (Object-Relational Mapping) to fetch data from internal APIs, ensuring efficient and dynamic content retrieval while maintaining SEO best practices.
